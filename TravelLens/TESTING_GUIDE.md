# 🧪 인앱결제 테스트 가이드

인앱결제 상품 등록이 완료되었으니 이제 테스트를 진행하세요!

---

## 📱 iOS 테스트 (Sandbox)

### 1. Sandbox 테스트 계정 생성

1. [App Store Connect](https://appstoreconnect.apple.com) 로그인
2. **사용자 및 액세스** → **샌드박스 테스터** 클릭
3. **+** 버튼으로 새 테스터 추가
   - 이메일: 실제 사용하지 않는 이메일 (예: test@example.com)
   - 비밀번호: 설정
   - 국가/지역: 테스트할 국가 선택

### 2. 테스트 기기 설정

1. **설정** → **App Store** → **Sandbox 계정** 로그아웃 (기존 계정이 있다면)
2. 앱 실행
3. 인앱결제 시도 시 Sandbox 계정으로 로그인 요청
4. Sandbox 계정으로 로그인

### 3. 테스트 시나리오

✅ **정상 구매 플로우**
1. 앱 실행
2. 홈 화면에서 "자세히" 버튼 클릭
3. 프리미엄 모달에서 "5일 패스 구매" 클릭
4. Sandbox 계정으로 로그인
5. 구매 확인
6. 모달이 자동으로 닫히는지 확인
7. 프리미엄 상태가 활성화되는지 확인

✅ **구매 복원**
1. 프리미엄 모달에서 "구매 복원" 버튼 클릭
2. 이전 구매가 복원되는지 확인

✅ **취소 테스트**
1. 구매 화면에서 "취소" 클릭
2. 에러 없이 정상적으로 돌아오는지 확인

## 🤖 Android 테스트

### 1. 테스트 계정 설정

1. Google Play Console에서 **설정** → **계정 세부정보**
2. 테스트 계정 추가 (또는 기존 Google 계정 사용)
3. **내부 테스트** 또는 **알파 테스트** 트랙에 추가

### 2. 테스트 기기 설정

1. Google Play Store 앱에서 테스트 계정으로 로그인
2. 앱 설치 (테스트 트랙에서)
3. 인앱결제 테스트

### 3. 테스트 시나리오

✅ **정상 구매 플로우**
1. 앱 실행
2. 홈 화면에서 "자세히" 버튼 클릭
3. 프리미엄 모달에서 "5일 패스 구매" 클릭
4. Google Play 결제 화면에서 결제 진행
5. 구매 완료 후 모달이 닫히는지 확인
6. 프리미엄 상태가 활성화되는지 확인

✅ **구매 복원**
1. 프리미엄 모달에서 "구매 복원" 버튼 클릭
2. 이전 구매가 복원되는지 확인

## 🐛 문제 해결

### 상품이 로드되지 않는 경우

**iOS:**
- App Store Connect에서 상품이 **승인됨** 상태인지 확인
- 상품 ID가 정확히 `kfinder_unlimited_search`인지 확인
- 프로덕션 빌드에서만 테스트 가능 (개발 빌드에서는 Sandbox 상품도 안 보일 수 있음)

**Android:**
- Google Play Console에서 상품이 **활성** 상태인지 확인
- 상품 ID가 정확히 `kfinder_unlimited_search`인지 확인
- 앱이 Google Play에서 설치된 버전인지 확인 (직접 설치한 APK는 인앱결제 불가)

### 구매가 완료되지 않는 경우

1. **로그 확인**
   ```javascript
   // 개발자 콘솔에서 확인
   console.log("구매 오류:", error);
   ```

2. **일반적인 원인**
   - 네트워크 연결 문제
   - 스토어 계정 문제
   - 상품 ID 불일치
   - 결제 수단 문제

### 구매 복원이 안 되는 경우

- 이전에 실제로 구매한 내역이 있어야 함
- 같은 Google/Apple 계정으로 로그인했는지 확인
- Firebase에 구매 정보가 저장되어 있는지 확인

## 🚀 프로덕션 테스트

### 실제 결제 테스트 (소액)

**⚠️ 주의**: 실제 결제가 발생합니다!

1. **소액으로 테스트**
   - 상품 가격을 최소 금액으로 설정 (예: $0.99)
   - 테스트 구매 후 환불 요청

2. **테스트 후**
   - 상품 가격을 원래대로 복구
   - 환불 처리 확인
