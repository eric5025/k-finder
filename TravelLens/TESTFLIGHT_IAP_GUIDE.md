# 🧪 TestFlight에서 인앱결제 테스트 가이드

TestFlight에 올라가 있는 앱에서도 인앱결제를 테스트할 수 있습니다!

---

## ✅ 핵심 정리

### ⚠️ 중요: iOS 첫 인앱결제 상품 등록

**iOS의 경우:**
- **첫 번째 인앱결제 상품**을 등록할 때는 **앱 심사와 함께 제출**해야 합니다
- 상품만 따로 검토용 제출하는 것이 아니라, 앱과 함께 심사에 제출해야 함
- 이후 추가 상품들은 독립적으로 검토 가능

### TestFlight에서 인앱결제 테스트 가능 여부

**답: 가능합니다!** ✅

- TestFlight 빌드에서도 인앱결제 테스트 가능
- **Sandbox 계정**을 사용하여 테스트
- 다만, 첫 인앱결제 상품은 앱 심사와 함께 제출해야 함

### 앱 심사 제출 vs 인앱결제 테스트

**앱 심사 제출:**
- App Store에 앱을 출시하기 위한 절차
- **첫 인앱결제 상품은 앱과 함께 심사 필요**

**인앱결제 테스트:**
- TestFlight에서도 가능 (앱 심사 후)
- Sandbox 계정 사용
- 인앱결제 상품이 등록되어 있어야 함

---

## 📋 TestFlight 인앱결제 테스트 조건

### 필수 조건

1. **인앱결제 상품 등록 완료**
   - App Store Connect에 상품 ID `kfinder_unlimited_search` 등록
   - iOS: 상품이 "승인됨" 또는 "검토 중" 상태
   - Android: 상품이 "활성" 상태

2. **TestFlight 빌드 업로드**
   - EAS Build로 빌드
   - TestFlight에 업로드

3. **Sandbox 테스트 계정**
   - App Store Connect에서 Sandbox 테스터 생성

---

## 🚀 TestFlight 인앱결제 테스트 방법

### 1단계: Sandbox 테스트 계정 생성

1. [App Store Connect](https://appstoreconnect.apple.com) 로그인
2. **사용자 및 액세스** → **샌드박스 테스터** 클릭
3. **+** 버튼으로 새 테스터 추가
   - 이메일: 실제 사용하지 않는 이메일
   - 비밀번호: 설정
   - 국가/지역: 테스트할 국가 선택

### 2단계: 기기에서 Sandbox 계정 설정

1. **설정** → **App Store** → **Sandbox 계정** 로그아웃 (기존 계정이 있다면)
2. TestFlight 앱 설치
3. TestFlight에서 앱 다운로드 및 설치

### 3단계: 인앱결제 테스트

1. TestFlight 앱 실행
2. 인앱결제 시도
3. 결제 화면에서 **Sandbox 계정으로 로그인** 요청
4. Sandbox 계정 정보 입력
5. 구매 진행

## ⚠️ 중요 사항

### iOS 인앱결제 상품 상태

**검토 중 (In Review):**
- ✅ Sandbox에서 테스트 가능
- ❌ 프로덕션에서는 사용 불가

**승인됨 (Approved):**
- ✅ Sandbox에서 테스트 가능
- ✅ 프로덕션에서도 사용 가능

**거부됨 (Rejected):**
- ❌ 테스트 불가
- 문제 수정 후 재제출 필요

### TestFlight vs 프로덕션 빌드

**TestFlight 빌드:**
- Sandbox 환경에서 테스트
- 실제 결제는 발생하지 않음
- Sandbox 계정 필요

**프로덕션 빌드 (App Store 출시 후):**
- 실제 결제 발생
- 실제 Apple ID 사용
- 실제 돈이 청구됨

## 🔍 문제 해결

### 상품이 보이지 않는 경우

**원인:**
- 인앱결제 상품이 등록되지 않음
- 상품 ID가 일치하지 않음
- 상품이 "거부됨" 상태

**해결:**
1. App Store Connect에서 상품 상태 확인
2. 상품 ID가 `kfinder_unlimited_search`인지 확인
3. 코드의 `PRODUCT_IDS`와 일치하는지 확인

### 구매가 안 되는 경우

**원인:**
- Sandbox 계정이 설정되지 않음
- 네트워크 문제
- 상품이 "승인됨" 상태가 아님

**해결:**
1. 설정 → App Store → Sandbox 계정 확인
2. Sandbox 계정으로 로그인
3. 상품 상태 확인

## 🎯 정리

### 질문: TestFlight에서 인앱결제 테스트 가능한가?

**답: 네, 가능합니다!** ✅

**조건:**
1. 인앱결제 상품이 App Store Connect에 등록되어 있어야 함
2. Sandbox 테스트 계정이 필요함
3. TestFlight 빌드가 업로드되어 있어야 함

**앱 심사 제출:**
- TestFlight 테스트와는 별개
- App Store 출시를 위한 절차
- 인앱결제 테스트는 심사 제출 전에도 가능

## 💡 권장 순서

### iOS 첫 인앱결제 상품의 경우

1. **인앱결제 상품 등록**
   - App Store Connect에서 상품 등록
   - 상품 정보 입력 완료

2. **앱 심사 준비**
   - 모든 필수 항목 채우기 (스크린샷, 설명, 개인정보처리방침 등)
   - TestFlight 빌드 업로드

3. **앱 심사 제출** (상품과 함께)
   - 앱과 인앱결제 상품을 함께 심사에 제출
   - Apple 검토 대기 (1-3일)

4. **검토 완료 후**
   - 상품이 "승인됨" 상태가 되면
   - TestFlight에서 Sandbox 계정으로 테스트 가능
   - 또는 App Store 출시 후 실제 사용 가능

### Android의 경우

- 인앱결제 상품은 즉시 활성화됨
- 앱 심사와 별개로 상품 등록 가능
- TestFlight 테스트도 독립적으로 가능

---

**iOS 첫 인앱결제 상품은 앱 심사와 함께 제출해야 하므로, 먼저 앱 심사를 준비하고 제출하세요! 🚀**
