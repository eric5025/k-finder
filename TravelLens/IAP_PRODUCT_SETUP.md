# 💳 인앱결제 상품 등록 가이드 (단계별)

인앱결제 상품을 등록하는 방법을 단계별로 안내합니다.

---

## 📱 iOS - App Store Connect

### 1단계: App Store Connect 접속

1. [App Store Connect](https://appstoreconnect.apple.com)에 로그인
2. **내 앱** 클릭
3. **Korea Finder** 앱 선택
   - 만약 앱이 없다면 먼저 앱을 생성해야 합니다

### 2단계: 인앱 구매 섹션으로 이동

1. 앱 상세 페이지에서 **기능** 탭 클릭
2. 왼쪽 사이드바에서 **인앱 구매** 클릭
3. **+** 버튼 클릭 (또는 "인앱 구매 만들기" 버튼)

### 3단계: 상품 유형 선택

**비소모성 상품** (Non-Consumable) 선택
- 한 번 구매하면 계속 사용할 수 있는 상품
- 우리 앱의 "5일 패스"에 적합

### 4단계: 상품 정보 입력

#### 필수 정보

1. **참조 이름** (Reference Name)
   - 예: `5일 검색 패스`
   - 내부 관리용 이름 (사용자에게 보이지 않음)

2. **상품 ID** (Product ID) ⚠️ **매우 중요!**
   - 입력: `kfinder_unlimited_search`
   - ⚠️ **코드와 정확히 일치해야 함!**
   - 대소문자 구분됨
   - 한 번 설정하면 변경 불가

3. **가격** (Price)
   - 원하는 가격 선택 (예: $4.99)
   - 지역별로 자동 변환됨

#### 표시 정보 (사용자에게 보이는 정보)

4. **표시 이름** (Display Name)
   - 한국어: `5일 검색 패스`
   - 영어: `5-Day Search Pass`
   - 다른 언어도 추가 가능

5. **설명** (Description)
   - 한국어: `5일 동안 무제한으로 검색할 수 있는 프리미엄 패스입니다.`
   - 영어: `Enjoy unlimited searches for 5 consecutive days.`

### 5단계: 저장 및 검토 제출

⚠️ **중요: 첫 번째 인앱결제 상품의 경우**

1. 모든 정보 입력 후 **저장** 클릭
2. **상품만 따로 검토용 제출하지 않습니다**
3. **앱 심사와 함께 제출**해야 합니다
4. 앱 심사 제출 시 상품도 함께 심사됨
5. Apple 검토 완료까지 **1-3일** 소요

**절차:**
- 인앱결제 상품 정보 입력 완료
- 앱 심사 준비 (스크린샷, 설명, 개인정보처리방침 등)
- **앱 심사 제출** 시 상품도 함께 제출됨

**이후 추가 상품:**
- 두 번째 상품부터는 독립적으로 검토 가능
- "검토용 제출" 버튼으로 별도 제출 가능

### 6단계: 상태 확인

상품 상태 확인:
- **검토 중** (In Review): Apple이 검토 중
- **승인됨** (Approved): 사용 가능 ✅
- **거부됨** (Rejected): 문제 수정 필요

## 🤖 Android - Google Play Console

### 1단계: Google Play Console 접속

1. [Google Play Console](https://play.google.com/console)에 로그인
2. 왼쪽 메뉴에서 **앱** 클릭
3. **Korea Finder** 앱 선택
   - 만약 앱이 없다면 먼저 앱을 생성해야 합니다

### 2단계: 인앱 상품 섹션으로 이동

1. 왼쪽 메뉴에서 **수익 창출** 클릭
2. **앱 내 상품** 클릭
3. **인앱 상품** 탭 클릭
4. **상품 만들기** 버튼 클릭

### 3단계: 상품 정보 입력

#### 필수 정보

1. **상품 ID** (Product ID) ⚠️ **매우 중요!**
   - 입력: `kfinder_unlimited_search`
   - ⚠️ **코드와 정확히 일치해야 함!**
   - 대소문자 구분됨
   - 한 번 설정하면 변경 불가

2. **이름** (Name)
   - 한국어: `5일 검색 패스`
   - 영어: `5-Day Search Pass`
   - 다른 언어도 추가 가능

3. **설명** (Description)
   - 한국어: `5일 동안 무제한으로 검색할 수 있는 프리미엄 패스입니다.`
   - 영어: `Enjoy unlimited searches for 5 consecutive days.`

4. **상태** (Status)
   - **활성** 선택 (즉시 사용 가능)

#### 가격 설정

5. **가격** (Price)
   - 원하는 가격 선택 (예: $4.99)
   - 지역별로 자동 변환됨
   - 또는 각 국가별로 개별 설정 가능

### 4단계: 저장 및 활성화

1. 모든 정보 입력 후 **저장** 클릭
2. 상품이 생성되면 **활성화** 버튼 클릭
3. ⚠️ **Android는 검토 없이 즉시 활성화됨!**

### 5단계: 확인

- 상품 목록에서 상태가 **활성**인지 확인
- 즉시 사용 가능 ✅

## ⚠️ 중요 체크리스트

### 상품 ID 확인

**코드에서 사용하는 상품 ID:**
```typescript
// 파일: src/hooks/useSearchLimit.ts
const PRODUCT_IDS = ["kfinder_unlimited_search"];
```

**스토어에 등록할 상품 ID:**
- ✅ `kfinder_unlimited_search` (정확히 일치해야 함!)

### 등록 전 확인

- [ ] 상품 ID가 코드와 정확히 일치하는가?
- [ ] 가격이 올바르게 설정되었는가?
- [ ] 표시 이름과 설명이 적절한가?

### 등록 후 확인

**iOS:**
- [ ] 상품이 생성되었는가?
- [ ] 검토용 제출을 했는가?
- [ ] 상태가 "검토 중" 또는 "승인됨"인가?

**Android:**
- [ ] 상품이 생성되었는가?
- [ ] 상태가 "활성"인가?

## 🧪 등록 후 테스트

### iOS 테스트

1. **Sandbox 계정 생성**
   - App Store Connect → 사용자 및 액세스 → 샌드박스 테스터
   - 테스트 계정 추가

2. **테스트**
   - 프로덕션 빌드에서 테스트
   - Sandbox 계정으로 로그인
   - 구매 시도

### Android 테스트

1. **테스트 계정**
   - Google Play Console에서 테스트 계정 추가
   - 또는 기존 Google 계정 사용

2. **테스트**
   - 프로덕션 빌드에서 테스트
   - 구매 시도

## 🐛 문제 해결

### 상품이 보이지 않는 경우

**iOS:**
- 상품이 "승인됨" 상태인지 확인
- 프로덕션 빌드에서 테스트 (개발 빌드에서는 안 보일 수 있음)
- 상품 ID가 정확한지 확인

**Android:**
- 상품이 "활성" 상태인지 확인
- Google Play에서 설치한 앱인지 확인 (직접 설치한 APK는 안 됨)
- 상품 ID가 정확한지 확인

### 구매가 안 되는 경우

1. **상품 ID 확인**
   - 코드와 스토어의 상품 ID가 정확히 일치하는지 확인
   - 대소문자 구분됨

2. **상품 상태 확인**
   - iOS: "승인됨" 상태여야 함
   - Android: "활성" 상태여야 함

3. **빌드 타입 확인**
   - 프로덕션 빌드에서만 실제 결제 가능
   - 개발 빌드에서는 제한적
