# 📱 iOS App Store Connect 제출 가이드

App Store Connect에서 앱 심사 제출을 위해 필요한 모든 항목을 채우는 가이드입니다.

---

## ✅ 필수 항목 체크리스트

### 1. 스크린샷 (필수)

#### iPhone 6.5인치 스크린샷
- **크기**: 1242 × 2688px 또는 2688 × 1242px 또는 1284 × 2778px 또는 2778 × 1284px
- **필요 개수**: 최소 3개 (최대 10개)
- **권장**: 5-8개

**촬영 방법:**
1. iPhone 14 Pro Max 또는 iPhone 15 Pro Max에서 앱 실행
2. 시뮬레이터 사용 시: Xcode → Window → Devices and Simulators → iPhone 14 Pro Max 선택
3. 각 화면에서 스크린샷 촬영:
   - 로그인 화면
   - 홈 화면 (카메라/갤러리 선택)
   - 검색 결과 화면
   - 검색 기록 화면
   - 프리미엄 모달 (선택)

#### iPad 13인치 스크린샷
- **크기**: iPad Pro 12.9인치 크기
- **필요 개수**: 최소 3개 (최대 10개)
- **권장**: 5개

**촬영 방법:**
1. iPad Pro 12.9인치 시뮬레이터 사용
2. 또는 실제 iPad에서 촬영
3. iPhone과 동일한 화면들 촬영

**팁:**
- 스크린샷은 앱의 주요 기능을 보여줘야 함
- 첫 번째 스크린샷이 가장 중요 (앱 설치 시트에 표시)
- 깔끔하고 전문적인 느낌

### 2. 앱 정보 (한국어)

#### 지원 URL (필수)
- **예시**: `https://yourwebsite.com/support` 또는 `https://github.com/yourusername/k-finder`
- **내용**: 사용자 지원 페이지
- **권장**: GitHub Pages, Netlify, Vercel 등에 호스팅

**간단한 지원 페이지 예시:**
```html
<!DOCTYPE html>
<html>
<head>
    <title>Korea Finder 지원</title>
</head>
<body>
    <h1>Korea Finder 지원</h1>
    <p>문의사항이 있으시면 이메일로 연락해주세요: your-email@example.com</p>
    <h2>자주 묻는 질문</h2>
    <h3>앱 사용 방법</h3>
    <p>카메라로 기념품을 촬영하거나 갤러리에서 선택하면 AI가 자동으로 인식합니다.</p>
    <!-- 추가 FAQ -->
</body>
</html>
```

#### 키워드 (필수)
- **최대 100자**
- **예시**: `한국,기념품,인식,AI,여행,관광,쇼핑,이미지검색,다국어`
- **팁**: 쉼표로 구분, 공백 없이

**추천 키워드:**
- 한국, 기념품, 인식, AI, 여행, 관광, 쇼핑, 이미지검색, 다국어, 번역, 한국여행, 기념품찾기

#### 설명 (필수)
- **최대 4,000자**
- **내용**: 앱의 기능, 특징, 사용 방법 등

**예시 설명:**
```
Korea Finder는 외국인 관광객을 위한 AI 기반 한국 기념품 인식 앱입니다.

✨ 주요 기능
• AI 이미지 인식: 사진을 찍으면 한국 기념품을 자동으로 인식합니다
• 다국어 지원: 한국어, 영어, 일본어, 중국어, 스페인어로 정보를 제공합니다
• 검색 기록: 이전에 검색한 기념품을 언제든지 다시 확인할 수 있습니다
• 프리미엄 패스: 5일 동안 무제한으로 검색할 수 있는 프리미엄 패스를 제공합니다

📸 사용 방법
1. 카메라로 기념품을 촬영하거나 갤러리에서 사진을 선택합니다
2. AI가 자동으로 기념품을 분석하고 정보를 제공합니다
3. 다국어로 번역된 상세 정보를 확인합니다

🎯 누구를 위한 앱인가요?
• 한국을 방문하는 외국인 관광객
• 한국 기념품에 관심이 있는 분들
• 쇼핑 중 기념품 정보가 필요한 분들

🔒 개인정보 보호
• Google 로그인 또는 익명 로그인 지원
• 검색 기록은 안전하게 저장됩니다
• 개인정보 처리방침을 준수합니다

지금 다운로드하고 한국 기념품을 더 쉽게 발견하세요!
```

### 3. 기본 카테고리 선택 (필수)

**추천 카테고리:**
- **주 카테고리**: 여행 (Travel)
- **보조 카테고리**: 쇼핑 (Shopping) 또는 유틸리티 (Utilities)

**선택 방법:**
1. App Store Connect → 앱 정보 → 카테고리
2. 주 카테고리와 보조 카테고리 선택

### 4. 연령 등급 (필수)

**추천 설정:**
- **연령 등급**: 4+ (모든 연령)
- **또는**: 9+ (9세 이상)

**콘텐츠 설명 빈도 수준:**
- **의료/치료 정보**: 없음
- **약물/담배/알코올**: 없음
- **도박/경쟁**: 없음
- **성적 콘텐츠/나체**: 없음
- **공포/공포물**: 없음
- **만화/판타지 폭력**: 없음
- **현실적 폭력**: 없음
- **프로파나/저속한 유머**: 없음
- **성적 콘텐츠 또는 나체**: 없음
- **불법 약물**: 없음
- **참고/교육 정보**: 없음 또는 드물게
- **경쟁**: 없음

**대부분 "없음" 또는 "드물게"로 설정하면 됩니다.**

### 5. 개인정보 처리방침 URL (필수)

**필요한 것:**
- 개인정보 처리방침 페이지를 웹에 호스팅
- URL을 App Store Connect에 입력

**호스팅 방법:**

**옵션 1: GitHub Pages (무료, 추천)**
1. GitHub 저장소 생성
2. `privacy-policy.html` 파일 생성
3. GitHub Pages 활성화
4. URL: `https://yourusername.github.io/repo-name/privacy-policy.html`

**옵션 2: Netlify (무료)**
1. Netlify 계정 생성
2. HTML 파일 업로드
3. 자동으로 URL 생성

**옵션 3: Vercel (무료)**
1. Vercel 계정 생성
2. 프로젝트 배포
3. URL 생성

**개인정보 처리방침 내용:**
- `PRIVACY_POLICY_TEMPLATE.md` 파일 참고
- 실제 정보로 수정 필요

**예시 URL:**
- `https://yourusername.github.io/k-finder/privacy-policy.html`
- `https://k-finder.netlify.app/privacy-policy.html`

### 6. 앱 심사 정보

#### 로그인 정보 (필수)
- **로그인 필요**: 예
- **사용자 이름**: 테스트 계정 이메일
- **암호**: 테스트 계정 비밀번호

**테스트 계정 생성:**
1. Firebase Console → Authentication
2. 사용자 추가 또는 Google 로그인 테스트 계정 사용
3. App Store Connect에 입력

**또는:**
- **로그인 필요**: 아니요 (익명 로그인 가능하므로)
- 익명 로그인으로도 사용 가능하다면 "아니요" 선택 가능

#### 연락처 정보 (필수)
- **이름**: 개발자 이름
- **성**: 개발자 성
- **전화번호**: 연락 가능한 전화번호
- **이메일**: 연락 가능한 이메일
- **메모**: (선택사항) 앱 심사팀을 위한 추가 정보

### 7. 기타 선택 사항

#### 프로모션 텍스트 (선택)
- **최대 170자**
- **내용**: 앱의 주요 특징을 간단히 소개
- **예시**: "AI로 한국 기념품을 즉시 인식! 다국어 지원으로 더 쉬운 한국 여행"

#### 마케팅 URL (선택)
- 앱 홍보 웹사이트
- 없으면 비워둬도 됨

#### 저작권
- **예시**: `© 2024 Your Name` 또는 `© 2024 Your Company`

## ⚠️ 주의사항

1. **개인정보 처리방침 URL**
   - 반드시 접근 가능한 URL이어야 함
   - 404 에러가 나면 거부될 수 있음

2. **스크린샷**
   - 실제 앱 화면과 일치해야 함
   - 가짜 스크린샷은 거부될 수 있음

3. **설명**
   - 과장된 표현 피하기
   - 실제 기능만 설명
